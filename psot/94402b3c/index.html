# 实例名，服务名

Oracle JDBC 连接一共有三种方式，分别是：SERVICE_NAME、SID 和 TNSName。

1.SERVICE_NAME 方式：jdbc:oracle:thin:@//<host>:<port>/<SERVICE_NAME>

2.SID 连接方式：jdbc:oracle:thin:@<host>:<port>:<SID>
或：jdbc:oracle:thin:@<host>:<port>/<SID>

3.TNSName 连接方式：jdbc:oracle:thin:@<TNSName>

打开 oracle 路径下的 D:\oraclexe\app\oracle\product\11.2.0\server\network\ADMIN\tnsames.ora 文件

红线框内的 db25 就是 TNSName，是属于客户端的参数，其余内容都是服务端的参数。

SERVICE_NAME 和 SID 的比较：
SID 是对内的，是实例级别的一个名字，用来内部之间称呼用。
SERVICE_NAME 是对外的，是数据库级别的一个名字，用来告诉外面的人，我数据库叫 "SERVICE_NAME"。

访问数据库的过程：
要想访问数据库，必须把数据库文件加载进实例中。SID 即 INSTANCE_NAME 是用来唯一标示实例的。SERVICE_NAME 是 oracle8i 新引进的，8i 之前，一个数据库只能由一个实例对应，但是随着高性能的需求，并行技术的使用，一个数据库可以由多个实例对应了，比较典型的应用如 RAC。为了充分利用所有实例，并且令客户端连接配置简单，ORACLE 提出了 SERVICE_NAME 的概念，该参数直接对应数据库，而不是某个实例。自此 Oracle JDBC 连接多使用 SERVICE_NAME 方式连接，逐渐替代 SID 方式连接。
